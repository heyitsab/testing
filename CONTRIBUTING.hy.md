# Ներդրման ուղեցույց

Շնորհակալություն մեր նախագծում ներդրում կատարելու ձեր հետաքրքրության համար։ Մենք ողջունում ենք ցանկացած ներդրում՝ սկսած սխալների զեկույցներից մինչև նոր հատկությունների ավելացում։

## Ինչպես նպաստել

### Սխալների զեկուցում

Եթե դուք գտել եք սխալ, խնդրում ենք բացել issue հետևյալ տեղեկատվությամբ՝

- Սխալի հստակ և համառոտ նկարագրություն
- Քայլեր՝ սխալը վերարտադրելու համար
- Ակնկալվող վարքագիծ
- Իրական վարքագիծ
- Էկրանի լուսանկարներ (եթե կարելի է)
- Ձեր միջավայրի մասին տեղեկություն (Ruby տարբերակ, օպերացիոն համակարգ)

### Կոդի փոփոխություններ

1. **Fork արեք պահոցը**
   ```bash
   # Սեղմեք "Fork" կոճակը GitHub-ում
   ```

2. **Կլոնավորեք ձեր fork-ը**
   ```bash
   git clone https://github.com/YOUR-USERNAME/testing.git
   cd testing
   ```

3. **Ստեղծեք նոր branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Կատարեք ձեր փոփոխությունները**
   - Գրեք պարզ և ընթեռնելի կոդ
   - Հետևեք Ruby ոճի ուղեցույցին
   - Ավելացրեք թեստեր ձեր փոփոխությունների համար
   - Թարմացրեք փաստաթղթերը, եթե անհրաժեշտ է

5. **Ստուգեք ձեր կոդը**
   ```bash
   ruby alpaca_test.rb
   ```

6. **Commit արեք ձեր փոփոխությունները**
   ```bash
   git add .
   git commit -m "Ավելացնել՝ [համառոտ նկարագրություն]"
   ```

7. **Push արեք ձեր branch-ը**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Ստեղծեք Pull Request**
   - Անցեք GitHub և բացեք Pull Request
   - Տրամադրեք հստակ նկարագրություն ձեր փոփոխությունների մասին
   - Հղում տվեք ցանկացած առնչվող issue-ների

## Կոդավորման ստանդարտներ

### Ruby ոճի ուղեցույց

- Օգտագործեք 2 բացատ ներդրման համար
- Օգտագործեք snake_case փոփոխականների և մեթոդների համար
- Օգտագործեք CamelCase դասերի համար
- Պահպանեք տողերը 80 նիշերի սահմանում
- Գրեք հստակ և նկարագրական անուններ

### Թեստավորում

- Բոլոր նոր ֆունկցիոնալությունները պետք է ունենան թեստեր
- Համոզվեք, որ բոլոր թեստերը անցնում են՝
  ```bash
  ruby alpaca_test.rb
  ```
- Մի օգտագործեք `puts` կամ `binding.pry` արտադրական կոդում

### Commit հաղորդագրություններ

Գրեք հստակ և նկարագրական commit հաղորդագրություններ՝

- Օգտագործեք ներկա ժամանակ ("Ավելացնել հատկություն" ոչ թե "Ավելացրել հատկություն")
- Սկսեք մեծատառով
- Պահպանեք առաջին տողը 50 նիշերի սահմանում
- Ավելացրեք մանրամասն նկարագրություն, եթե անհրաժեշտ է

Օրինակներ՝
```
Ավելացնել feed մեթոդ Alpaca դասին

Այս մեթոդը թույլ է տալիս alpaca-ին կերակրել
տարբեր սնունդ տեսակներով։
```

## Վարքագծի կանոններ

### Մեր խոստումը

Մենք պարտավորվում ենք ապահովել բաց և ընդունող միջավայր բոլորի համար, անկախ՝
- Տարիքից
- Մարմնի չափից
- Հաշմանդամությունից
- Էթնիկությունից
- Սեռային նույնականությունից և արտահայտումից
- Փորձի մակարդակից
- Ազգությունից
- Անձնական արտաքին տեսքից
- Ռասայից
- Կրոնից
- Սեռական նույնականությունից և կողմնորոշումից

### Մեր ստանդարտները

Դրական միջավայր ստեղծող վարքագիծը ներառում է՝

- Հարգալից և ներառական լեզվի օգտագործում
- Տարբեր տեսակետների և փորձի հարգում
- Կառուցողական քննադատության նրբագեղ ընդունում
- Համայնքի լավագույնի վրա կենտրոնացում
- Համայնքի այլ անդամների նկատմամբ համակրանք ցուցաբերում

### Կիրարկում

Անընդունելի վարքագծի դեպքերը կարելի է զեկուցել նախագծի պահպանողներին։ Բոլոր բողոքները կքննարկվեն և կհետաքննվեն։

## Հարցեր

Եթե ունեք հարցեր, խնդրում ենք՝

- Բացեք issue քննարկման համար
- Կապ հաստատեք պահպանողների հետ

Շնորհակալություն ձեր ներդրման համար։ Մենք գնահատում ենք ձեր ժամանակը և ջանքերը։
