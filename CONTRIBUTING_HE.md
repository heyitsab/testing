## קוד התנהגות

אנא התנהגו בכבוד ובאדיבות כלפי כל התורמים. אנו שואפים ליצור סביבה פתוחה ומזמינה לכולם.

## איך אפשר לתרום?

ישנן דרכים רבות לתרום לפרויקט:

- 🐛 דיווח על באגים
- 💡 הצעת פיצ'רים חדשים
- 📝 שיפור התיעוד
- 🔧 תיקון באגים
- ✨ הוספת פיצ'רים חדשים
- 🧪 כתיבת בדיקות
- 📖 תרגום

## תהליך הגשת שינויים

1. **Fork את הפרויקט**
   ```bash
   # לחצו על כפתור Fork ב-GitHub
   ```

2. **שכפלו את ה-Fork שלכם**
   ```bash
   git clone https://github.com/YOUR-USERNAME/testing.git
   cd testing
   ```

3. **צרו ענף חדש**
   ```bash
   git checkout -b feature/amazing-feature
   ```

4. **בצעו את השינויים שלכם**
   - ודאו שהקוד עומד בסטנדרטים
   - הוסיפו בדיקות אם רלוונטי
   - עדכנו תיעוד אם נדרש

5. **הריצו את הבדיקות**
   ```bash
   ruby alpaca_test.rb
   ```

6. **בצעו Commit לשינויים**
   ```bash
   git add .
   git commit -m "הוספת פיצ'ר מדהים"
   ```

7. **דחפו לענף שלכם**
   ```bash
   git push origin feature/amazing-feature
   ```

8. **פתחו Pull Request**
   - גשו ל-GitHub ופתחו Pull Request
   - תארו בבירור את השינויים שביצעתם
   - הוסיפו צילומי מסך אם רלוונטי

## סטנדרטים לקוד

### Ruby

- השתמשו ב-Ruby idiomatic code
- עקבו אחר Ruby Style Guide
- וודאו שהקוד קריא ומתועד היטב
- הימנעו משימוש ב-`puts` בקוד הייצור (ראו `alpaca_test.rb`)
- הימנעו משימוש ב-`binding.pry` בקוד הסופי

### כללי

- שמרו על עקביות עם סגנון הקוד הקיים
- כתבו קוד נקי וקריא
- הוסיפו הערות כשצריך, אך העדיפו קוד מדבר מעצמו
- וודאו שכל הבדיקות עוברות

## הודעות Commit

כתבו הודעות commit ברורות ותמציתיות:

- השתמשו בזמן הווה ("הוספת פיצ'ר" לא "הוספתי פיצ'ר")
- התחילו עם אות רישית
- שמרו על השורה הראשונה קצרה (50 תווים או פחות)
- הוסיפו תיאור מפורט אם נדרש

דוגמאות טובות:
```
הוספת מתודה לחישוב משקל צמר
תיקון באג בהצגת מידע אלפקה
עדכון תיעוד עם דוגמאות שימוש
```

## דיווח על באגים

כשמדווחים על באג, אנא כללו:

- **תיאור ברור** של הבעיה
- **שלבי שחזור** - איך לשחזר את הבאג
- **התנהגות צפויה** - מה היה אמור לקרות
- **התנהגות בפועל** - מה קרה בפועל
- **סביבה** - גרסת Ruby, מערכת הפעלה וכו'
- **צילומי מסך** - אם רלוונטי

## הצעת פיצ'רים חדשים

יש לכם רעיון לפיצ'ר חדש? נשמח לשמוע!

כשמציעים פיצ'ר, אנא כללו:

- **תיאור ברור** של הפיצ'ר
- **נימוק** - למה הפיצ'ר הזה שימושי
- **דוגמאות שימוש** - איך זה ייראה בפועל
- **חלופות שונות** - האם שקלתם גישות אחרות
